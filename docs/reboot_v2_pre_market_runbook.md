# Reboot V2 Pre-Market Operations Runbook

This runbook defines the default Reboot V2 / Theme Core V3 observe startup path. It does not add a new strategy, tune thresholds, enable DRY_RUN/LIVE_SIM, or allow REAL orders.

## Default Startup

Use this as the default 64-bit Core startup script:

```powershell
.\tools\start_reboot_v2_observe.ps1
```

The script starts `trading_app.api:app` with these safety defaults:

```text
STRATEGY_RUNTIME_PROFILE=THEME_CORE_V3
STRATEGY_REBOOT_V2_PROFILE=THEME_CORE_V3
STRATEGY_REBOOT_V2_ENABLED=1
TRADING_RUNTIME_ENABLED=1
TRADING_RUNTIME_AUTO_START=1
TRADING_RUNTIME_MODE=OBSERVE
TRADING_THEME_CORE_V3_ENABLED=1
TRADING_THEME_CORE_V3_INGEST_CANDIDATES=0
TRADING_OPENING_BURST_ENABLED=1
TRADING_OPENING_BURST_CONFIGURED=1
TRADING_DASHBOARD_V2_ENABLED=1
TRADING_DASHBOARD_V2_AUTO_ROUTE=1
TRADING_ORDER_MANAGER_ENABLED=0
TRADING_ALLOW_LIVE_SIM_ORDERS=0
TRADING_ENTRY_ALLOW_DRY_RUN_INTENTS=0
TRADING_EXIT_ALLOW_DRY_RUN_SELL_INTENTS=0
```

The 32-bit Kiwoom Gateway remains a separate process and connects to the Core URL. Gateway command polling must remain observe-safe: no `send_order` command should be generated by this profile.

## Legacy Rollback

Legacy runtime is explicit only:

```powershell
$env:STRATEGY_RUNTIME_PROFILE="LEGACY"
```

Legacy is frozen for rollback/debug. It is the only profile that may construct the old `GatePipeline`, `ThemeLabRuntimePipeline`, `EntryPlanBuilder`, legacy exit engine, legacy virtual order service, shadow/promotion provider, or threshold A/B runtime hooks.

The old `tools/start_market_open_live_sim.ps1` path is deprecated for normal operation. Treat it as a legacy/live-sim runbook path, not as the default Reboot V2 opening flow.

## Default Routes

- `/`: Dashboard V2
- `/legacy`: frozen legacy ThemeLab dashboard
- `/themelab`: compatibility alias for legacy ThemeLab dashboard
- `/debug`: raw Core/system diagnostics

## Pre-Market Checklist

Before 09:00:

- confirm Core starts with `STRATEGY_RUNTIME_PROFILE=THEME_CORE_V3`
- confirm Dashboard V2 is visible at `/`
- confirm `/api/dashboard-v2/snapshot` returns V2 schema
- confirm `/api/gateway/commands` has zero `send_order` commands
- confirm `TRADING_ORDER_MANAGER_ENABLED=0`
- confirm `TRADING_ALLOW_LIVE_SIM_ORDERS=0`
- confirm `TRADING_THEME_CORE_V3_INGEST_CANDIDATES=0`
- confirm legacy dashboard is reachable only through `/legacy` or `/themelab`

## Opening Observe Window

During 09:00-09:15 the runtime observes Opening Burst + Theme Core V3 outputs only:

- opening turnover seeds
- realtime theme leadership
- candidate bridge source events in observe mode
- market regime
- entry/exit observe summaries
- Dashboard V2 leading theme and wait/block summaries

Watchset, candidate, and READY-style concepts must not create Gateway `send_order` commands in this profile.

## Forbidden Changes

- Do not enable DRY_RUN intent generation in this cutover.
- Do not enable LIVE_SIM orders in this cutover.
- Do not add REAL order flags.
- Do not create Gateway `send_order` commands.
- Do not tune thresholds or theme scores.
- Do not route condition-search include events directly to READY or order intents.
